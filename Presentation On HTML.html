  1. HISTORY OF HTML

    BY EMMANUEL MENSAH

    1. What is HTML?

    2. HTML Timeline

    3. HTML 1.0

      .HTML 1.0 was released in 1993
      .With the intention of sharing information that can be readable and accessible via web browsers.
      - But not many of the developers were involved in creating websites
      . So the language was also not growing.
      - The code was simple with not enough tags to create a good website

    4. HTML 2.0

      .HTML 2.0 was published in 1995
      .which contains all the features of HTML 1.0 along with that few additional features.
      -which remained as the standard markup language for designing and creating websites until January 1997.
      . Refined various core features of HTML.
      - This was the official standard language.

  3. HTML 3.0 & 3.2

    Then comes the HTML 3.0, where Dave Raggett who introduced a fresh paper or draft on HTML. It included improved new features of HTML, giving more powerful characteristics for webmasters in designing web pages. But these powerful features of new HTML slowed down the browser in applying further improvements. So it was abandoned. W3C was created in 1994 to standardize the language and keep it evolving in the right direction. Then by 1996 Html 3.2 was created.

    1. HTML 4.0

      Html 4.0 was invented in 1997 and became the official standard in 1998.

      Html 4.0 was used by many browser for a long time.

      Later HTML 4.01 was revised and corrected. It was the final version of the specification.

    2. XHTML 1.0

      Almost close to the 21st century the W3C introduced the XHTML 1.0 spec.

      The only difference from the other spec was that they added a new set of coding rules.

      Writing XHTML required one to follow rules relevant to XML, such as correct syntax and structure.

  5. HTML 5.0

    The last version of HTML was done when most of the W3C were focus on building XHTML 2.0.

    So other programmers came together to form a group called WHATWG.

    Which stands for Web Hypertext Application Technology Working Group.

    When W3C heard about it they stopped with XHTML 2.0 and HTML5 became the latest version. It had no flaws.

    It was designed for the web, both now and the future.

    2. ADVANTAGES DISADVANTAGES
      HTML is widely used. HTML can create only static and plain pages so if we need dynamic pages then HTML is not useful
      Every browser supports HTML Language. I need to write a lot of code for making a simple webpage.
      Easy to learn and use. And you do not need to purchase any extra software because it is by default in every window. Security features are not good at HTML. And if we need to write long code for making a webpage then it produces some complexity.

    3. OPINION

      To my understand html doesn’t work on its own because it is not a programming language but a markup language. It is a markup language because it allows the user to create and structure sections, paragraphs, headings, links, and block quotes for web pages and applications. HTML is not a programming language, meaning it doesn't have the ability to create dynamic functionality.
